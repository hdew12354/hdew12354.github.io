Tweet-Time

'insert michael reeves "do you ever realise you are making a bad idea" quote meme here'

So, In this project I Wanted a way of telling people 'Hey, it's super late, my brain's probably not functioning in the most coherant way right now, take whatever i'm saying with a grain of salt' and I don't think many people take the time stamp of a tweet into consideration when they're eciding if they like a tweet or note so I decided to develop a more 'in your face' way of telling people the time of a tweet being posted in the poster's time-zone.

To do this I needed 3 things, those being

  • A Server, be it home built or online, such as AWS Lambda
  • Twitter App Developer Credentials, which are avalible to the public for free
  • A Custom written Program to finally enact the changes to the account that I needed

So, Now that I had credentials, i can get started with the programming aspect of this project. for this, i decided to use Python in conjunction with the Tweepy Module to speak with the Twitter API. I decided to use Python because i am most familliar with it, due to me having learned it in high school for my Computer Science GCSE, because I am well acquainted with the interface and because it allows me to work on the project whislt at college.

Now that i knew what i was going to run the app in, that being python, i needed to think up a way of telling the reader the time, and so i decided to use the username of the tweeter, by adding a small string onto the username, containing the time zone and time that the tweet was posted, as you can see below!

How I decided to have the application function is pretty simple actually, all the app does is constantly run a function that asks the twitter API 'hey, how many tweets has this user put out?' and stores what the API says back, it then waits for 5 seconds to allow for the user to post another tweet, and then asks again, if the new ammount is higher than the old ammount the app goes 'hey this is higher! there must be a new tweet!' it then asks the API what the user's time zone is set to, calculates the time in that time zone and then asks the API what the account name is. this is where the little half p, half b looking abomination character(Ãž), called a Thorn comes into play, the application splits the username into 2 parts at the thorn, giving it the pure, un-edited username, it then adds the thorn character back onto the username as well as the user's timezone and the time there! it then finally tells the API to set the username to the newly made username that it just made!
